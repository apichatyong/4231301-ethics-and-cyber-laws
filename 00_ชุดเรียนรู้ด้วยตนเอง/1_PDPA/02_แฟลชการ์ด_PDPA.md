# 🔑 แฟลชการ์ด PDPA (ถาม–ตอบ)
### วิธีใช้: ปิดฝั่งขวา (คำตอบ) แล้วตอบในใจ/เขียน → เปิดเช็ค ทำซ้ำจนตอบได้ทุกใบ

> เน้นตัวเลขมาตรา + ตัวเลขสำคัญ (ออกปรนัยบ่อย)

| # | คำถาม | คำตอบ |
|---|---|---|
| 1 | PDPA คือ พ.ร.บ. อะไร ปีใด | พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล **พ.ศ. 2562** |
| 2 | บังคับใช้เต็มรูปแบบเมื่อใด | **1 มิถุนายน 2565** |
| 3 | หน่วยงานกำกับ PDPA | สำนักงานคณะกรรมการคุ้มครองข้อมูลส่วนบุคคล (**สคส./PDPC**) |
| 4 | มาตราที่นิยาม "ข้อมูลส่วนบุคคล" | **มาตรา 6** |
| 5 | นิยามข้อมูลส่วนบุคคล | ข้อมูลที่ระบุตัวบุคคลได้ ทางตรงหรือทางอ้อม **ไม่รวมผู้ถึงแก่กรรม** |
| 6 | มาตรา "ข้อยกเว้นไม่ใช้บังคับ" | **มาตรา 4** (6 กรณี) |
| 7 | มาตราขอบเขตบังคับใช้ (ใน/นอกประเทศ) | **มาตรา 5** |
| 8 | DC ย่อมาจาก / แปลว่า | Data Controller = ผู้มีอำนาจ **ตัดสินใจ** เก็บ/ใช้/เปิดเผยข้อมูล |
| 9 | DP ย่อมาจาก / แปลว่า | Data Processor = ผู้ดำเนินการ **ตามคำสั่ง** DC |
| 10 | DPO คือใคร อยู่มาตราใด | เจ้าหน้าที่คุ้มครองข้อมูลส่วนบุคคล (**ม.41–42**) |
| 11 | PDPA มีกี่ฐานการประมวลผล | **7 ฐาน** (Consent ม.19 + 6 ข้อยกเว้น ม.24) |
| 12 | ฐานความยินยอม อยู่มาตราใด | **มาตรา 19** |
| 13 | ฐานสัญญา (Contract) | **ม.24(3)** |
| 14 | ฐานหน้าที่ตามกฎหมาย (Legal Obligation) | **ม.24(6)** |
| 15 | ฐานระงับอันตรายต่อชีวิต (Vital Interest) | **ม.24(2)** [Sensitive: ม.26(1)] |
| 16 | ฐานภารกิจรัฐ (Public Task) | **ม.24(4)** |
| 17 | ฐานประโยชน์โดยชอบด้วยกฎหมาย (Legitimate Interest) | **ม.24(5)** + ต้องทำ Balancing Test |
| 18 | ฐานวิจัย/สถิติ (Research) | **ม.24(1)** |
| 19 | กล้องวงจรปิด (CCTV) ใช้ฐานใด | ฐานประโยชน์โดยชอบด้วยกฎหมาย **ม.24(5)** |
| 20 | ถอนความยินยอมได้เมื่อใด มาตราใด | **เมื่อใดก็ได้** (ม.19 ว.5) |
| 21 | ความยินยอมต้องทำรูปแบบใด | ชัดแจ้ง เป็นหนังสือ/อิเล็กทรอนิกส์ แยกส่วน ไม่หลอกลวง |
| 22 | ผู้เยาว์อายุเท่าใดต้องให้ผู้ปกครองยินยอมเสมอ | **ไม่เกิน 10 ปี** |
| 23 | ข้อมูลอ่อนไหว (Sensitive) อยู่มาตราใด | **มาตรา 26** |
| 24 | ยกตัวอย่างข้อมูลอ่อนไหว 5 อย่าง | เชื้อชาติ, ศาสนา, สุขภาพ, พฤติกรรมทางเพศ, ประวัติอาชญากรรม, พันธุกรรม, ชีวภาพ, สหภาพแรงงาน, ความเห็นการเมือง, ความพิการ |
| 25 | Sensitive Data ต้องขอความยินยอมแบบใด | **ความยินยอมโดยชัดแจ้ง (Explicit Consent)** |
| 26 | สิทธิเข้าถึง & ขอสำเนา | Right of Access — **ม.30** |
| 27 | สิทธิโอนย้ายข้อมูล | Data Portability — **ม.31** |
| 28 | สิทธิคัดค้าน | Right to Object — **ม.32** |
| 29 | สิทธิขอลบ/ทำลาย | Right to Erasure / to be forgotten — **ม.33** |
| 30 | สิทธิระงับการใช้ | Right to Restriction — **ม.34** |
| 31 | สิทธิขอแก้ไขให้ถูกต้อง | Right to Rectification — **ม.36** |
| 32 | สิทธิได้รับการแจ้ง / Privacy Notice | **ม.23** |
| 33 | หน้าที่ DC อยู่มาตราใด | **มาตรา 37** |
| 34 | มาตรการความปลอดภัยครอบคลุม 3 ด้านอะไร | **CIA** = Confidentiality, Integrity, Availability |
| 35 | ต้องแจ้งเหตุละเมิดข้อมูลต่อ สคส. ภายในกี่ชั่วโมง | **72 ชั่วโมง** นับแต่ทราบ (ม.37(4)) |
| 36 | DP แจ้งเหตุละเมิดต่อใคร | ต่อ **DC** (ไม่ใช่ สคส.) — ม.40 |
| 37 | ROPA คืออะไร มาตราใด | Record of Processing Activities — บันทึกรายการประมวลผล (**ม.39**) |
| 38 | หน้าที่ DP อยู่มาตราใด | **มาตรา 40** |
| 39 | การโอนข้อมูลไปต่างประเทศ มาตราใด | **มาตรา 28** — ปลายทางต้องมีมาตรฐานคุ้มครองเพียงพอ |
| 40 | โทษอาญาสูงสุดของ PDPA | จำคุก ≤ 1 ปี และ/หรือ ปรับ ≤ **1,000,000 บาท** |
| 41 | โทษทางปกครองสูงสุด | ปรับ ≤ **5,000,000 บาท** |
| 42 | ค่าสินไหมเพื่อการลงโทษ สูงสุดกี่เท่า | ไม่เกิน **2 เท่า** ของค่าสินไหมที่แท้จริง (ม.77–78) |
| 43 | Anonymization ต่างจาก Pseudonymization อย่างไร | Anonymization = ระบุตัวไม่ได้ **พ้น**จากเป็นข้อมูลส่วนบุคคล; Pseudonymization = แฝงข้อมูล **ยังเป็น**ข้อมูลส่วนบุคคล |
| 44 | ฐานรัฐธรรมนูญของสิทธิความเป็นส่วนตัว | รัฐธรรมนูญ 2560 **มาตรา 32** |
| 45 | เป้าหมายสูงสุดเชิงจริยธรรมของ PDPA | สร้าง **ความไว้วางใจ (Trust)** ในสังคม/เศรษฐกิจดิจิทัล |
| 46 | 5 เสาหลักจริยธรรม PDPA | ศักดิ์ศรีมนุษย์, ความเป็นส่วนตัว, การกำหนดเจตนาตนเอง, ความเป็นธรรม/ยุติธรรม, ความรับผิดชอบ/โปร่งใส |
| 47 | กฎหมายต่างชาติที่เป็นต้นแบบ PDPA | GDPR (EU), PDPA 2012 (สิงคโปร์) |
| 48 | บทเฉพาะกาลเรื่องข้อมูลเดิม | **มาตรา 95** |

---

### ⚡ การ์ดดักหลุมพราง (ข้อสอบชอบหลอก)
- ❌ "ต้องขอความยินยอมเสมอ" → **ผิด** มี 6 ฐานอื่นที่ไม่ต้องขอ
- ❌ "ข้อมูลคนตายเป็นข้อมูลส่วนบุคคล" → **ผิด** ม.6 ไม่รวมผู้ถึงแก่กรรม
- ❌ "แจ้งเหตุละเมิดภายใน 24 ชม." → **ผิด** คือ **72 ชม.**
- ❌ "DP แจ้งเหตุละเมิดต่อ สคส." → **ผิด** DP แจ้งต่อ **DC**
- ❌ "ฐานสัญญาใช้กับข้อมูลอ่อนไหวได้" → **ผิด** Sensitive ต้องใช้ฐาน ม.26
